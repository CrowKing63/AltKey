- 레이아웃에서 컨트롤 + 알트 + 쉬프트 + O 키 콤보 저장하면 다음 실행 때 컨트롤 + C + C + A로 저장이 돼 있음. 기대한 기능(앞의 맵핑)이 작동하는 것 같기는 한데 컨트롤 A 기능도 실행됨.
- 키보드 레이아웃 편집기에서 새로운 키를 추가하고 저장할 경우, 적용 중인 레이아웃에 변동 없음. 다른 레이아웃으로 바꿨다가 다시 돌아와도 저장 전 레이아웃. 프로그램을 완전히 껐다가 켜야만 변경 사항이 적용됨.
- 레이아웃의 키 설정에서 드롭다운의 가상 키 코드 라벨만을 보고 원하는 키를 찾기가 너무 힘들다. 일반적인 키 이름으로 보여줘야 한다.
- 현재 각 버튼마다 너비와 앞 간격을 지정할 수 있다. 그런데 컨트롤 키나 스페이스 키 등의 키들의 너비가 제각각인 상태에서 좌측에 새로운 키를 각 행마다 추가해서 사용자만의 열을 추가하고 싶을 때 해당 열을 일렬로 맞추기가 매우 힘들다. 현재는 새로운 행만 추가할 수 있는데, 열 추가도 가능하게 해서 기존 열과 간격을 일정하게 격리시키는 식으로 구현이 될까?
- 행의 끝에 버튼을 추가해도 키보드 전체 창을 수동으로 키워줘야 하는 번거로움이 있다. 

- 자동 설치 및 재시작된 프로그램이 관리자 권한으로 실행된다. 자동 완성과 유니코드 인풋을 위해서는 프로그램을 완전히 끄고 다시 켜는 번거로움이 있다.
- 다른 컴퓨터 환경에서 키보드 입력이 무수히 작동하며 그동안 다른 물리적 입력이 불가해지는 버그 출현. 하지만 내 환경에서는 재현 불가. 그런데 키 반복 입력을 사용할 때도 비슷한 소리가 나서 그쪽부터 조사를 해 봐야 할 듯. 그리고 그 기능은 기본값을 오프로 수정. 스티키 키는 무조건 반복 입력 안 되게. 윈도우 키도 컨트롤, 알트, 쉬프트 키처럼 스티키 키로 수정.
- 자동 완성 기능을 써서 글을 쓰다보면 낮은 빈도로 이전 단어와의 공백(스페이스 한칸)을 씹고 붙은 채 입력되고는 함. 아직 조건은 불명확.
