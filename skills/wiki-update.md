# Skill: 위키 문서 업데이트 및 게시 (Wiki Update & Publish)

이 스킬은 기능을 추가, 변경 또는 삭제했을 때 `Wiki/AltKey.wiki`의 관련 매뉴얼을 수정하고, 변경 사항을 위키 Git 저장소에 반영(커밋/푸시)하기 위한 통합 지침입니다.

## 목표
최근 코드 변경 사항을 분석하여 사용자 조작 방법, 설정 위치, 안내 문구 등의 변경점을 위키 문서에 정확히 반영하고 원격 저장소(`origin/master`)에 즉시 게시합니다.

## 실행 단계

### 1. 최근 변경 사항 분석 (Analyze Recent Changes)
업데이트가 필요한 내용을 파악하기 위해 메인 저장소의 최근 변경 내역을 검토합니다.
1. **커밋 내역 확인**: `git log -n 5 --oneline` 명령으로 최근 작업 내역을 확인합니다.
2. **변경 상세 분석**: 필요 시 `git show [COMMIT_ID]` 또는 `git diff HEAD~1`을 통해 코드와 설정 파일(`AppConfig.cs`, `Basic.json` 등)의 변화를 상세히 분석합니다.
3. **영향도 평가**: 다음 질문을 통해 위키 업데이트 여부를 결정합니다.
   - 새로운 기능이 추가되었는가?
   - 기존 기능의 조작 방식이나 단축키가 변경되었는가?
   - 설정 화면의 UI 문구나 옵션 위치가 바뀌었는가?
   - 접근성 관련 기능(L1/L3 모드 등)에 변화가 있는가?

### 2. 위키 문서 식별 및 수정
1. **대상 식별**: `docs/wiki-meta-manual.md`의 "§6 소스 매핑" 섹션을 참조하여, 변경된 소스 파일과 관련된 위키 페이지를 찾습니다.
2. **문서 수정**:
   - `Wiki/AltKey.wiki/{페이지이름}.md` 파일을 열어 실제 변경 사항에 맞게 내용을 수정합니다.
   - `docs/wiki-meta-manual.md`의 템플릿과 규칙(한국어 작성, 스크린샷 자리 표시자 등)을 준수합니다.
   - 관련 페이지의 링크나 Home(메인 페이지)의 소개 문구도 함께 검토하여 필요시 수정합니다.

### 3. 위키 변경 사항 확인 (Git)
- `git -C "C:\Users\UITAEK\AltKey\Wiki\AltKey.wiki" status --short` 명령으로 위키 저장소의 수정·추가된 파일 목록을 최종 확인합니다.
- 변경 사항이 없으면 작업을 종료합니다.

### 4. 커밋 및 게시 (Commit & Publish)
1. **커밋 메시지 작성**: 한국어로 다음 형식을 따릅니다.
   - 제목: `docs: [변경 내용 한 줄 요약]`
   - 본문(선택): 수정된 문서 목록 또는 주요 변경 내용
2. **스테이징 및 커밋**:
   - `git -C "C:\Users\UITAEK\AltKey\Wiki\AltKey.wiki" add .`
   - `git -C "C:\Users\UITAEK\AltKey\Wiki\AltKey.wiki" commit -m "[제목]" -m "[본문]"`
3. **원격 푸시**:
   - `git -C "C:\Users\UITAEK\AltKey\Wiki\AltKey.wiki" push origin master`
   - 푸시 성공 여부를 확인합니다.

## 주의 사항
- 기능을 **추가/변경/삭제**한 경우, 같은 작업(Task) 안에서 반드시 위키 매뉴얼도 함께 갱신해야 합니다.
- 위키 저장소는 메인 프로젝트(`AltKey/`)와 별개의 Git 저장소이므로 모든 git 명령에 `-C` 옵션으로 경로를 명시해야 합니다.
- PowerShell 환경이므로 `&&` 대신 `;` 또는 순차 실행을 사용합니다.
