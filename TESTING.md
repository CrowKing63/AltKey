# AltKey — 최종 통합 테스트 체크리스트

> 배포 전 포터블 빌드에서 전 기능을 수동으로 검증합니다.  
> 모든 항목을 `[x]`로 표시한 후 릴리즈를 진행하세요.

## 환경 준비

1. `scripts/package-portable.ps1 -Version 0.1.0` 실행 → `dist/altkey-portable-v0.1.0.zip` 생성
2. ZIP 압축 해제 → 새 폴더에서 `AltKey.exe` 실행 (개발 환경 아님)

---

## 체크리스트

### 기본 동작

- [x] 포터블 exe 실행 → Acrylic 배경 키보드 표시
- [x] 메모장 열고 키 클릭 → 입력 전달 (포커스 유지 확인)

### 수정자 키 (Modifier)

- [x] Shift 단일 클릭 → 다음 키 대문자 입력 → Shift 자동 해제
- [x] Shift 더블클릭 → 영구 잠금(Caps) → 재클릭으로 해제
- [x] 한/영 키 클릭 → IME 전환 (한국어 ↔ 영어)

### 창 관리

- [x] 드래그 핸들로 창 이동 → 앱 종료 → 재시작 → 위치 유지
- [x] 리사이즈 그립으로 창 크기 변경 → 최소 크기(400×200) 이하로 줄어들지 않음
- [x] 마우스가 창을 벗어난 후 5초 → 창 반투명 → 마우스 재진입 → 불투명 복귀

### 단축키 및 트레이

- [x] `Ctrl+Alt+K` → 창 숨기기/보이기 토글 (다른 앱이 포그라운드여도 동작)
- [x] 앱 최소화 버튼 클릭 or 닫기 → 트레이 이동 (태스크바 미표시) → 의도된 동작: WS_EX_TOOLWINDOW + ShowInTaskbar="False"로 태스크바에 아이콘 표시 안됨 (정상)
- [x] 트레이 아이콘 표시 확인
- [x] 트레이 우클릭 → 컨텍스트 메뉴(보이기/숨기기, 레이아웃, 설정, 종료) 동작
- [x] 트레이 아이콘 더블클릭 → 창 복귀

### 테마 및 레이아웃

- [x] 설정 패널(⚙) → 다크/라이트 테마 전환 → 즉시 반영
- [x] 상단 레이아웃 드롭다운 → `qwerty-en` 전환 → 키 레이블 변경

### 체류 클릭 (Dwell Click)

- [x] 설정에서 체류 클릭 활성화
- [x] 키 위에 마우스를 800ms 이상 올려 두면 → 입력 발생 + 프로그레스 링 표시 → 수정 완료 (DwellRing ZIndex 및 크기 조정)

### 에러 처리

- [ ] 잘못된 JSON의 레이아웃 파일 배치 → 경고 다이얼로그 표시 + `altkey-error.log` 기록 + 기본 레이아웃으로 폴백
- [ ] 관리자 권한 앱(예: 작업 관리자) 포그라운드 상태에서 키 클릭 → 권한 경고 배너 표시

### 포터블 빌드

- [ ] .NET 8 미설치 환경(VM 또는 클린 PC)에서 포터블 exe 실행 성공
- [x] ZIP 내 `config.json` 존재 → 포터블 모드로 동작 (사용자 데이터를 exe 옆에 저장)

### 업데이트 알림 (T-6.4)

- [x] `AssemblyVersion`을 낮춰 빌드 후 실행 → GitHub 최신 릴리즈보다 낮으면 상단 배너 표시
- [x] 배너 "다운로드" 클릭 → 브라우저에서 GitHub Release 페이지 열림
- [x] 배너 "✕" 클릭 → 배너 닫힘

## 성능 목표 (T-6.6)

| 항목 | 목표 | 측정 방법 |
|------|------|----------|
| 유휴 메모리 | < 50 MB | 작업 관리자 → 프로세스 탭 → 현재 250MB (WPF Acrylic/투명도 사용 시 정상 범위, 개선 필요) |
| 키 입력 지연 | < 10 ms | Debug 로그 Stopwatch 확인 |
| 앱 시작 시간 | < 2 초 | Debug 로그 `[AltKey] Startup time:` 확인 |

## 디버그 로그 확인 방법

1. **Visual Studio 사용**: `F5` 디버그 시작 → 출력 창에서 `[AltKey]` 로그 확인
2. **DebugView 도구**: [Sysinternals DebugView](https://learn.microsoft.com/sysinternals/downloads/debugview) 다운로드 → `Debug.WriteLine()` 출력 실시간 확인
3. **로그 파일**: 오류 발생 시 `altkey-error.log` 파일 확인 (exe 동일 디렉토리)
