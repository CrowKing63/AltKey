# 접근성 보강 수동 검증 체크리스트

## 목적

스위치 제어, 탭 네비게이션, 자동완성 접근성 보강 후 실제 사용 흐름이 깨지지 않았는지 사람이 직접 확인한다.

## 공통 준비

- 일반 권한 실행과 관리자 권한 실행 차이를 기록한다.
- 자동완성 OFF와 ON을 모두 확인한다.
- 테마는 라이트, 다크, 고대비를 각각 확인한다.
- 키 라벨 크기 100%, 150%, 200%를 각각 확인한다.
- 스캔 간격은 300ms, 800ms, 1500ms를 각각 확인한다.

## 일반 입력 회귀

- [ ] 마우스 클릭으로 한글 단어를 입력할 수 있다.
- [ ] 마우스 클릭으로 백스페이스, 스페이스, 엔터를 입력할 수 있다.
- [ ] `가/A` 토글 후 QuietEnglish 입력이 가능하다.
- [ ] 자동완성 OFF에서 기존 OS IME 기반 입력 흐름이 깨지지 않는다.
- [ ] 자동완성 ON에서 한글 조합 중 깜빡임, 중복 입력, 전체 삭제가 없다.

## 탭 네비게이션

- [ ] 탭 네비게이션 OFF에서는 물리 `Tab`, `Enter`, `Space`를 AltKey가 가로채지 않는다.
- [ ] 탭 네비게이션 ON에서 `Tab`으로 다음 키로 이동한다.
- [ ] `Shift+Tab`으로 이전 키로 이동한다.
- [ ] `Enter` 또는 `Space`로 현재 하이라이트 키를 실행한다.
- [ ] `Esc` 또는 설정된 탈출 키로 접근성 포커스를 해제할 수 있다.
- [ ] 자동완성 제안 포함 범위가 켜져 있으면 제안 단어로도 이동할 수 있다.
- [ ] 헤더 버튼 포함 범위가 켜져 있으면 설정, 접기/펼치기, 닫기 등으로 이동할 수 있다.

## 1스위치 자동 스캔

- [ ] 스위치 스캔 ON에서 자동으로 하이라이트가 이동한다.
- [ ] 선택 키로 현재 키를 입력할 수 있다.
- [ ] 선택 후 짧은 일시정지가 적용되어 같은 키가 실수로 반복 입력되지 않는다.
- [ ] 스캔 범위 끝에서 처음으로 돌아가는 설정이 의도대로 동작한다.
- [ ] 전체 순환 제한을 설정했을 때 지정 횟수 뒤 멈춘다.

## 2스위치/수동 스캔

- [ ] 다음 키로 하이라이트를 한 칸 이동할 수 있다.
- [ ] 이전 키가 설정된 경우 한 칸 뒤로 이동할 수 있다.
- [ ] 선택 키로 현재 대상을 실행할 수 있다.
- [ ] 수동 모드에서는 자동 타이머 이동이 발생하지 않는다.
- [ ] 일시정지 키가 설정된 경우 스캔 일시정지/재개가 가능하다.

## 입력 장치 커스텀

- [ ] 기본값 `Tab`, `Enter`, `Space`는 기존처럼 동작한다.
- [ ] 선택 키를 다른 `VirtualKeyCode`로 바꿔도 동작한다.
- [ ] 잘못된 키 이름을 넣어도 앱이 멈추지 않는다.
- [ ] 스위치 스캔 OFF에서는 커스텀 키를 AltKey가 가로채지 않는다.

## 자동완성 제안 스캔

- [ ] 자동완성 ON에서 제안 단어가 표시된다.
- [ ] 스캔 하이라이트가 제안 단어로 이동한다.
- [ ] 제안 단어 선택 시 마우스 클릭과 같은 결과로 수락된다.
- [ ] 현재 조합 단어 슬롯을 선택하면 저장 동작이 실행된다.
- [ ] 제안 선택 직후 다음 자모를 입력해도 이전 단어가 통째로 삭제되지 않는다.
- [ ] QuietEnglish 모드에서도 영어 제안을 선택할 수 있다.
- [ ] 자동완성 OFF에서는 제안 스캔 대상이 생기지 않는다.

## 공지와 TTS

- [ ] TTS OFF에서도 입력 기능은 동일하게 동작한다.
- [ ] TTS ON에서 키 선택 후 라벨이 읽힌다.
- [ ] LiveRegion 공지 ON에서 Narrator가 현재 위치를 읽는다.
- [ ] 빠른 스캔에서 같은 라벨이 과도하게 반복 낭독되지 않는다.
- [ ] 공지 모드 `Off`, `SelectionOnly`, `EveryMove`가 구분되어 동작한다.

## 시각적 확인

- [ ] 하이라이트가 라이트 테마에서 충분히 보인다.
- [ ] 하이라이트가 다크 테마에서 충분히 보인다.
- [ ] 하이라이트가 고대비 테마에서 충분히 보인다.
- [ ] 큰 글자 200%에서도 버튼 라벨이 겹치거나 UI 밖으로 튀지 않는다.
- [ ] Reduced Motion ON에서 하이라이트 이동과 입력이 어지럽지 않다.

## 최종 회귀 시나리오

- [ ] 한글 단어 입력 → 자동완성 제안 선택 → 스페이스 → 다음 단어 입력
- [ ] 한글 조합 중 백스페이스 여러 번 → 빈 필드까지 정상 삭제
- [ ] `가/A` 전환 → 영어 입력 → 자동완성 제안 선택 → 한글 모드 복귀
- [ ] 스위치 스캔 ON/OFF 반복 후 하이라이트가 남지 않는다.
- [ ] 탭 네비게이션 ON/OFF 반복 후 물리 키 가로채기 상태가 정상 복귀한다.
