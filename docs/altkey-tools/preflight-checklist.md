# 착수 전 체크리스트: AltKey.Tools 1단계

## 목적

이 문서는 실제 코드 작업을 시작하기 직전에 확인해야 할 항목을 정리한다.

목표는 다음 두 가지다.

- 작업 도중 범위가 다시 커지는 것을 막는다.
- 메인 입력 기능과 접근성 기능에 대한 회귀 위험을 초기에 통제한다.

## 1. 범위 확인

아래 항목이 모두 만족되어야 1단계 착수를 시작한다.

- [ ] 이번 단계의 목표가 `설정 앱 전체 분리`가 아니라 `편집 도구 앱 생성`임이 팀 내에서 명확하다.
- [ ] 1차 분리 대상이 `레이아웃 편집기`, `사용자 단어 편집기`임이 확정되었다.
- [ ] `프로필 매핑 편집`은 1단계 필수 범위가 아니라 검토 또는 후순위임이 명확하다.
- [ ] 전역 단축키, 스위치 스캔, TTS, 창 크기, 상단바 버튼, 업데이트는 메인 앱 유지 대상임이 합의되었다.

## 2. 구조 확인

- [ ] `AltKey.Tools`는 별도 WPF 실행 파일로 추가한다.
- [ ] 메인 앱은 편집 대상 데이터 본문을 IPC로 보내지 않는다.
- [ ] 메인 앱과 도구 앱은 같은 사용자 데이터 폴더를 읽는다.
- [ ] 1단계 IPC는 재로드 알림 수준으로 제한한다.
- [ ] 자동완성/한글 조합 핵심 구조는 이번 단계에서 수정하지 않는다.

## 3. 레이아웃 편집기 선행 확인

- [ ] `LayoutEditorViewModel`의 핵심 의존성이 `LayoutService`, `ConfigService` 중심임을 확인했다.
- [ ] `LayoutEditorWindow`의 `FocusTracker.Register(this)` 제거 또는 대체 계획이 있다.
- [ ] 저장 후 메인 앱에 `ReloadLayouts`를 보낼 계획이 있다.
- [ ] 기본 레이아웃 삭제 제한 정책을 도구 앱에서도 유지할 방법이 있다.

## 4. 사용자 단어 편집기 선행 확인

- [ ] `UserDictionaryEditorViewModel`의 핵심 의존성이 `KoreanDictionary`, `EnglishDictionary`, `WordFrequencyStore`, `BigramFrequencyStore`임을 확인했다.
- [ ] `OnLoaded()` / `OnClosing()` 수명주기 처리를 도구 앱에서 재현할 계획이 있다.
- [ ] `FocusTracker.Register(this)` 제거 또는 대체 계획이 있다.
- [ ] 저장/삭제 후 메인 앱에 `ReloadUserDictionary`, `ReloadBigramData`를 보낼 계획이 있다.

## 5. 접근성 확인

- [ ] 도구 호스트 창의 첫 포커스 위치 규칙을 정했다.
- [ ] 레이아웃 편집기와 사용자 단어 편집기의 탭 순서 검증 방식을 정했다.
- [ ] `AutomationProperties` 유지 또는 보강 대상 컨트롤을 별도로 점검할 계획이 있다.
- [ ] `Esc` 닫기 동작 유지 여부를 결정했다.
- [ ] 메인 앱과 도구 앱 사이 창 전환 시 포커스 유실 여부를 수동 검증할 계획이 있다.

## 6. 검증 확인

- [ ] 레이아웃 편집 저장 후 메인 앱 반영 확인 절차가 있다.
- [ ] 사용자 단어 추가/삭제 후 자동완성 반영 확인 절차가 있다.
- [ ] 메인 앱 빌드와 테스트를 어떤 시점에 돌릴지 정했다.
- [ ] 수동 접근성 검증 항목이 별도 문서 또는 작업 메모에 준비되어 있다.

## 7. 착수 금지 조건

아래 중 하나라도 해당되면 실제 작업에 바로 들어가지 않는다.

- [ ] `AltKey.Tools`와 메인 앱의 책임 경계가 다시 모호해졌다.
- [ ] 1단계에서 설정창 전체까지 옮기자는 요구가 다시 포함되었다.
- [ ] 자동완성/한글 조합 핵심 코드 수정이 필요하다는 전제가 생겼다.
- [ ] 저장 후 메인 앱 재반영 방식이 합의되지 않았다.

## 요약

이 체크리스트가 모두 충족되면, 다음 작업은 `AltKey.Tools 프로젝트 뼈대 생성`부터 시작하면 된다.
