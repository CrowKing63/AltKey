# 퓨처 플랜: 레이아웃 편집기 액션 할당

## 개요
현재 AI 기능은 사용 편의성과 구현의 단순화를 위해 **상단바(Header Bar)의 ✨ 버튼**을 통해서만 실행할 수 있습니다. 
하지만 추후 사용자가 특정 키보드 버튼 자체에 AI 액션을 할당하고, 각 버튼마다 다른 프롬프트를 지정할 수 있도록 확장할 여지가 있습니다. (예: 어떤 버튼은 "요약", 어떤 버튼은 "번역")

## 구현 시 고려사항 (체크리스트)

1. **`KeyAction.cs` 확장에 따른 UI 반영**
   - 이미 `AiAction(string Prompt = "")` 레코드가 모델 단에 정의되어 있습니다.
   - 레이아웃 편집기(`LayoutEditorWindow.xaml` 및 `LayoutEditorViewModel.cs`) 내 액션 종류 선택(ComboBox 등)에 "AI 프롬프트 실행"을 추가해야 합니다.
   
2. **프롬프트 입력 UI 제공**
   - 편집기에서 `AiAction`을 선택했을 때, 해당 버튼이 수행할 **프롬프트를 입력할 수 있는 텍스트 박스**가 동적으로 노출되어야 합니다.
   - 이 프롬프트는 `AiAction.Prompt`에 저장되며, 비어있을 경우 `AppConfig.AiDefaultPrompt`를 사용하도록 로직을 설계합니다.

3. **`KeyboardViewModel.cs`의 키 처리 로직 확장**
   - `OnKeyPressed` 메서드 내부의 `KeyAction` 분기(switch)에 `AiAction` 처리를 추가해야 합니다.
   - 현재 상단바 버튼에서 사용되는 `MainViewModel.ExecuteAiCommand`를 호출하거나, 동일한 `AiService` 로직을 재사용하여 프롬프트 파라미터만 오버라이드하여 전달하도록 합니다.

## 시나리오
- 사용자는 `AltKey`의 레이아웃 편집기를 열어 자주 쓰는 단축키 버튼 대신 새로운 빈 버튼을 만들고, "AI 영문 번역"이라고 라벨을 붙입니다.
- 버튼 동작을 "AI 액션"으로 선택하고 프롬프트 란에 "다음 텍스트를 자연스러운 영어로 번역해줘." 라고 적습니다.
- 키보드를 띄운 뒤 아무 텍스트나 드래그하여 선택하고 해당 버튼을 누르면, 텍스트가 번역된 결과로 대체(Replace)됩니다.
