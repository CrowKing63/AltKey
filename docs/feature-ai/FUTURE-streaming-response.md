# 퓨처 플랜: 스트리밍 응답 (Streaming Response) 지원

## 스트리밍 응답이란?
AI 모델(특히 LLM)은 답변을 한 번에 생성하여 완성된 텍스트 전체를 보내는 대신, 토큰(단어 조각) 단위로 순차적으로 생성합니다. 
스트리밍 응답을 활성화하면 AI가 문장을 한 글자씩 타이핑하듯 실시간으로 결과를 받아볼 수 있습니다. 

**장점**: 사용자가 긴 응답을 기다리는 동안 앱이 멈춰있는 것처럼 느껴지는 것을 방지하고, 읽는 즉시 피드백을 받을 수 있습니다.
**단점**: 클립보드 기반의 현재 "Replace" 아키텍처(Ctrl+C -> API -> Ctrl+V)와는 잘 맞지 않습니다. 

## 현재 아키텍처와의 한계점
현재 AltKey의 AI 기능은 사용자가 텍스트를 드래그한 상태에서 버튼을 누르면:
1. `Ctrl+C`를 전송해 텍스트 복사
2. `HttpClient`로 전체 텍스트 전송 및 100% 완료된 응답 수신 대기 (블로킹/비동기 대기)
3. 응답 텍스트를 클립보드에 넣고 `Ctrl+V` 전송

만약 이 과정에 스트리밍을 도입해서 타겟 애플리케이션 창에 한 글자씩 `SendInput`을 날리거나 `Ctrl+V`를 여러 번 날릴 경우, 타겟 프로그램의 렌더링 성능 문제나 입력 포커스 유실, 클립보드 락(Lock) 경합 등 심각한 버그를 유발할 수 있습니다.

## 향후 구현 방안

스트리밍 응답을 도입하려면 클립보드 기반 입력을 우회하는 **독립적인 미리보기(Preview) UI 패널**이 필수적입니다.

1. **미리보기 패널 추가**: 상단바의 AI 버튼을 누르면, 가상 키보드 위에 작은 텍스트 패널이 나타납니다.
2. **스트리밍 수신**: AI의 응답이 이 패널에 실시간으로 타이핑되어 나타납니다. (OpenAI API의 `stream=true` 파라미터 사용, SSE(Server-Sent Events) 파싱)
3. **수동 붙여넣기(또는 자동 확정)**: 텍스트 생성이 완료된 후 사용자가 "적용" 버튼을 누르면 최종 텍스트가 한 번에 클립보드를 통해 타겟 앱으로 `Ctrl+V` 되는 구조입니다.

이 접근법은 복잡도를 높이지만(UI 렌더링 추가, 스트림 파싱 등), 사용자 경험(UX) 측면에서는 가장 권장되는 형태입니다.
