# Phase 1: 핵심 윈도우 관리

> 목표: AltKey 창이 포커스를 뺏지 않고, 항상 위에 떠 있으며, 드래그 이동과 투명도 제어가 가능한 상태를 만든다.

**의존성**: Phase 0 완료

---

## T-1.1: Win32 선언 정리

**설명**: 포커스 비침해, 항상 위, 창 위치 복원에 필요한 Win32 선언을 `Platform/Win32.cs`에 모은다.

**핵심 항목**
- `GetWindowLong`, `SetWindowLong`
- `SetWindowPos`
- `WS_EX_NOACTIVATE`
- `HWND_TOPMOST`, `HWND_NOTOPMOST`

**검증**: `dotnet build` 성공

---

## T-1.2: MainWindow 기본 구성

**설명**: 타이틀바 없는 메인 창과 투명 배경 기반 키보드 레이아웃을 준비한다.

**요점**
- `WindowStyle="None"`
- `ShowActivated="False"`
- `Topmost="True"`
- `Background="Transparent"`

**검증**: 앱 실행 시 타이틀바 없는 키보드 창이 정상 표시됨

---

## T-1.3: 포커스 비침해 적용

**설명**: 메인 키보드 창이 대상 앱 포커스를 빼앗지 않도록 `WS_EX_NOACTIVATE`를 적용한다.

**구현 메모**
- `OnSourceInitialized`에서 HWND 확보 후 적용
- WPF 속성만으로 부족한 부분은 Win32로 보완

**검증**: 메모장 등 대상 입력 창의 커서가 유지된 상태에서 AltKey 버튼 클릭 가능

---

## T-1.4: 배경 투명도와 폴백

**설명**: 화려한 시스템 배경 효과 대신, 입력면 가독성을 해치지 않는 투명도와 단색 폴백을 사용한다.

**원칙**
- 장식 효과보다 입력 가독성을 우선
- 유휴 상태에서만 투명도를 낮춤
- 환경에 따라 가독성이 떨어지면 단색 배경으로 폴백

**검증**: 사용 중/유휴 상태 모두 키 라벨과 배경 대비가 충분함

---

## T-1.5: 드래그 이동

**설명**: 상단 핸들 영역을 통해 메인 창을 쉽게 이동할 수 있게 한다.

**검증**: 핸들 드래그로 창 이동 가능, 키 영역 오조작 없음

---

## T-1.6: 창 위치 저장/복원

**설명**: 종료 시 창 위치를 저장하고 다음 실행 시 복원한다.

**검증**: 재실행 후 이전 위치로 복원, 화면 밖 좌표면 안전 위치로 재배치

---

## T-1.7: 자동 투명도

**설명**: 마우스 이탈 후 일정 시간이 지나면 유휴 투명도로 전환하고, 다시 들어오면 원래 상태로 복귀한다.

**검증**
1. 마우스 이탈 후 설정된 시간 경과 시 유휴 투명도 적용
2. 마우스 재진입 시 즉시 원래 불투명도로 복귀

---

## T-1.8: 항상 위 설정

**설명**: 사용자 설정에 따라 항상 위 여부를 바꿀 수 있게 한다.

**검증**: 설정 변경 시 즉시 반영되고 재실행 후에도 유지

---

## 완료 기준

- 포커스 비침해 동작 정상
- 드래그 이동 정상
- 창 위치 저장/복원 정상
- 자동 투명도 정상
- 항상 위 설정 정상
- 장식 효과 없이도 가독성과 접근성이 유지됨
