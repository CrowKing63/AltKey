# AltKey 프로젝트 주석 작성 가이드

이 문서는 접근성 측면에서, 코딩 전문가가 아니더라도 AltKey의 코드를 직접 읽고, 사소한 수정을 안전하고 편하게 할 수 있도록 돕기 위한 주석 작성 규칙을 정의합니다.

## 1. 기본 원칙
- **이해 중심**: 코드가 "무엇"을 하는지뿐만 아니라, **"왜"** 이렇게 작성되었는지를 설명합니다.
- **수정 편의성**: 하드코딩된 값(숫자, 색상 코드, 문자열 등) 옆에는 해당 값을 수정했을 때 어떤 변화가 생기는지 명시합니다.
- **친절한 언어**: 전문 용어를 최소화하고, 누구나 이해할 수 있는 쉬운 표현을 사용합니다.

## 2. 주석 형식 (최소 가이드라인)

### A. 클래스 및 파일 상단 (무엇을 위한 파일인가?)
파일의 가장 첫 부분이나 클래스 선언 바로 위에 작성합니다.
```csharp
/// <summary>
/// [역할] 키보드의 개별 버튼을 화면에 그리고 클릭 이벤트를 처리하는 클래스입니다.
/// [참고] 버튼의 크기나 폰트를 바꾸고 싶다면 이 클래스의 변수들을 확인하세요.
/// </summary>
public class KeyButton { ... }
```

### B. 메서드/함수 상단 (무엇을 하는 기능인가?)
함수가 시작되기 전 `///` 형식을 사용하여 작성합니다.
```csharp
/// <summary>
/// 사용자가 키를 눌렀을 때 입력될 텍스트를 결정합니다.
/// </summary>
/// <param name="keyCode">눌린 키의 고유 번호</param>
/// <returns>화면에 입력될 최종 문자열</returns>
public string GetInputText(int keyCode) { ... }
```

### C. 코드 내부 및 변수 (어디를 고치면 되는가?)
사소한 수정을 위해 코드 중간중간에 작성합니다. 특히 숫자가 들어가는 부분은 필수입니다.
```csharp
// 버튼 사이의 간격 (단위: 픽셀). 숫자를 키우면 버튼 사이가 멀어집니다.
double buttonSpacing = 5.0; 

// 한글/영어 전환 시 걸리는 지연 시간. 반응이 느리다고 느껴지면 이 값을 줄이세요.
int toggleDelayMs = 100; 
```
